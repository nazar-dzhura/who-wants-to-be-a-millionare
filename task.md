### Завдання:

Потрібно реалізувати базовий функціонал гри «Хто хоче стати мільйонером»: 

- Гравець по черзі відповідає на одне з 12 питань.
- Кожне питання має мінімум 4 варіанти відповіді і мінімум одну правильну.
- Якщо відповідь правильна - гравець потрапляє на наступне питання. Якщо відповідь неправильна - гравець потрапляє на фінальний екран.
- На фінальному екрані потрібно показати загальний результат гри. Дизайн для верстки
[надається](https://www.figma.com/file/tIZEZn2HTAeSDQRzoOzvXE/Front-end-test%2C-Headway). Верстка повинна бути адаптивною.

### Технічні вимоги:

- Використовувати цей [макет](https://www.figma.com/file/tIZEZn2HTAeSDQRzoOzvXE/Front-end-test%2C-Headway) для верстки.
- Зробити верстку адаптивною (від iPhone 8 до 4к дисплеїв).
- Верстку виконати без використання CSS фреймворків.
- Конфіг гри (питання, відповіді, гроші за правильну відповідь і.т.д.) повинен бути в форматі json.
- Конфіг повинен бути розширюваний: більше чи менше варіантів відповідей на питання; кілька правильних відповідей та ін.
- Результат роботи розмістити на github з md-інструкцією по установці і запуску.
- Зробити завдання з використанням NextJs та задеплоїти на Vercel.
- Весь клієнтський код повинен повністю відповідати правилам "з коробки" eslint / airbnb.
- Весь клієнтський код зробити типізованим через TypeScript.

### Опціонально:

- На git commit всередині репозиторія тестового завдання додати запуск eslint на змінені файли, а на git push запуск unit-tests.

### Критерії оцінки:

- Код який буде нескладно розширювати і підтримувати.
- Архітектурна організація застосунку.
- Коректна обробка позаштатних ситуацій.
